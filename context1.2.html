1.2 - Algorithms

1.2.1 - Understand what algorithms are and how they are expressed

Algorithms: An algorithm is a step-by-step procedure or formula for solving a problem. It is a well-defined sequence of actions or instructions that leads to a solution. Algorithms are used in all aspects of computing, from simple tasks like adding two numbers to complex processes like data encryption. They are fundamental to computer science because they help in automating tasks, solving problems efficiently, and optimizing processes.

Algorithms can be expressed in a variety of formats, such as:

  • Natural Language: Describing an algorithm in everyday language, though not as precise as other formats, it is often used for simple problems or when explaining concepts.
  • Flowcharts: A diagrammatic representation of an algorithm, showing the sequence of operations and decision points.
  • Pseudocode: A more structured, language-independent way of writing algorithms using plain language with a syntax similar to programming languages.

Example of an algorithm in pseudocode:

            1. Start
            2. Input number1, number2
            3. Calculate sum = number1 + number2
            4. Output sum
            5. End

1.2.2 - Be able to express an algorithm using flowcharts and pseudocode

Flowcharts and pseudocode are important tools for representing algorithms in a structured way:

  • Flowcharts: These are visual diagrams used to represent the sequence of operations in an algorithm. They use various symbols to denote different actions, such as ovals for start and end, rectangles for processes, diamonds for decision points, and arrows to show the flow of control. Flowcharts are helpful for understanding the logic and sequence of steps.
  • Pseudocode: Pseudocode is a way of expressing an algorithm in plain language using simple, structured statements. It helps to focus on the logic without worrying about specific programming syntax. For example, a simple pseudocode for finding the largest of two numbers might look like this:
    •                 1. Start
                2. Input number1, number2
                3. If number1 > number2 then
                4.     Output number1 is larger
                5. Else
                6.     Output number2 is larger
                7. End

1.2.3 - Be able to write algorithms that make use of programming constructs

When writing algorithms, it is essential to understand and use basic programming constructs, such as:

  • Sequence: This is the straightforward, step-by-step execution of operations. It is the simplest form of an algorithm, where actions occur one after another.
  • Selection: This involves decision-making, where different actions are taken based on a condition. For example, using an IF statement to check if a condition is true or false and taking appropriate actions based on that decision.
  • Iteration: Iteration is the repetition of steps or actions. It can be achieved using loops such as FOR or WHILE loops, which repeat a set of instructions as long as a condition holds true.

Example of an algorithm with iteration:

            1. Start
            2. Set counter to 1
            3. While counter <= 5 do
            4.     Output counter
            5.     Increment counter by 1
            6. End

1.2.4 - Understand the purpose of a given algorithm

The purpose of an algorithm is to solve a specific problem or perform a defined task. Each algorithm is designed with a goal in mind, such as sorting a list of numbers, searching for an item in a collection, or computing a mathematical result. Understanding the purpose of an algorithm helps in selecting or designing the most efficient solution. The purpose could also determine which algorithm to use (e.g., bubble sort vs. quicksort for sorting a list of numbers).

1.2.5 - Be able to determine the correct output of an algorithm

To determine the correct output of an algorithm, you must first understand the input and the steps it takes to process that input. You can verify the output by:

  • Manually tracing the algorithm: Walk through the algorithm step by step with sample inputs to verify that it produces the expected output.
  • Testing with different inputs: Run the algorithm with a variety of inputs, including edge cases (e.g., empty inputs, maximum values) to ensure robustness.

For example, if you want to find the sum of two numbers, the algorithm will take the two numbers as input and return their sum. Testing this with inputs like (3, 4) should give an output of 7.

1.2.6 - Be able to identify and correct errors in an algorithm

Errors in algorithms can arise due to incorrect logic, misunderstandings of the problem, or issues with the inputs. Here’s how you can identify and correct errors:

  • Test the algorithm: Run the algorithm with different inputs, including edge cases, to identify situations where it doesn’t behave as expected.
  • Trace the algorithm: Go through the algorithm step by step, manually or using a debugger, to identify where the logic deviates from the expected path.
  • Review pseudocode or flowchart: Sometimes the error becomes clear when looking at the algorithm in a structured format, as this may highlight overlooked conditions or incorrect logic.
  • Modify the algorithm: After identifying the error, adjust the logic, conditions, or flow to correct the algorithm. Re-test to verify that the correction works for all cases.

Example of an error correction:

If the algorithm for calculating the largest of two numbers is incorrectly written as:

            1. Start
            2. If number1 < number2 then
            3.     Output number1 is larger
            4. Else
            5.     Output number2 is larger
            6. End

It would output "number1 is larger" when number1 is smaller than number2. The error can be corrected by changing the condition in line 2 to number1 > number2.

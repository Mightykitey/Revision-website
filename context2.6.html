2.6.1 - Input Validation Techniques

Input validation is crucial to ensure that data entered into the system is correct, secure, and does not compromise the program's functionality. Common techniques include:

  • Presence Check: Ensures that the input is not empty, ensuring required fields are filled.
  • Length Check: Ensures that the input is within a valid range of acceptable lengths (e.g., password length must be between 8 and 16 characters).
  • Type Check: Verifies that the input corresponds to the expected data type (e.g., ensuring a phone number is numeric).
  • Format Check: Ensures the input matches a specific format, such as checking if an email address follows the correct pattern (e.g., example@domain.com).
  • Range Check: Verifies that the input value falls within an acceptable range (e.g., a temperature input between -50 and 50 degrees Celsius).
  • Check Digit: Used for numeric codes like credit card numbers or ISBNs, to verify their validity based on a checksum (e.g., Luhn algorithm for credit cards).

2.6.2 - Developing Reliable and Robust Code

To develop reliable, robust, and error-free code, consider the following practices:

  • Handle Edge Cases: Consider extreme or unexpected inputs when designing your code. These edge cases are often where bugs are found (e.g., handling division by zero, empty user input, or unusual characters).
  • Use Error Handling: Implement error handling mechanisms to gracefully manage unexpected issues. For example, in Python, the try and except blocks can be used to catch and handle exceptions, preventing the program from crashing.
  • Test Thoroughly: Write unit tests and functional tests to check your code for correctness. Ensure that all code paths are tested, including edge cases and failure conditions.
  • Logging: Use logging to capture important system information, errors, and events. This helps you debug problems that occur in production and track the performance and behavior of your program over time.

Example of Error Handling in Python:

    try:
        age = int(input("Enter your age: "))
        print("Your age is:", age)
    except ValueError:
        print("Oops! That wasn't a valid number.")

In the example above, the program attempts to convert the user input to an integer. If the user inputs something that is not a valid number (e.g., a string), the ValueError exception is raised, and the program will print an error message instead of crashing.

Example of Input Validation in Python:

    def validate_email(email):
        if "@" not in email or "." not in email:
            return False
        return True

    email = input("Enter your email: ")
    if validate_email(email):
        print("Email is valid.")
    else:
        print("Invalid email format.")

This example shows a simple email validation function that checks for the presence of both "@" and "." to ensure the input is a basic valid email address.

2.5.1 - Benefits and Drawbacks of Using Pre-Written Code

Using pre-written code such as built-in functions and standard libraries can save time and improve efficiency. However, there are some considerations:

  • Benefits:
    • Saves time by avoiding writing common functionality from scratch.
    • Well-tested and optimized for performance, ensuring reliability and speed.
    • Improves code readability and maintainability by using standard, recognizable functions.
  • Drawbacks:
    • Over-reliance on built-in functions may make the code harder to customize or adapt for specific needs.
    • Some built-in functions may not be suitable for all needs or environments, especially in specialized scenarios.
    • Can increase program size unnecessarily if unused functions or libraries are imported into the project.

2.5.2 - Selecting and Using Pre-Written Code (e.g., Built-in Functions, Standard Libraries)

Choosing appropriate built-in functions and libraries depends on the task at hand:

  • For mathematical tasks: Use the math module, which provides functions like math.sqrt() for square roots, math.factorial() for factorials, and many more.
  • For string manipulation: Use Python's built-in string functions like upper() to convert a string to uppercase or replace() to substitute substrings.
  • For working with collections: Python provides built-in data structures such as lists, sets, and dictionaries, with powerful methods like append() for lists or get() for dictionaries.
  • For file operations: The open() function is a built-in function used to read from or write to files, making file handling straightforward.

2.5.3 - Writing Code with User-Written and Pre-Written Code

Combining user-written and pre-written code allows for more flexible and efficient program development. Pre-written code provides standard solutions for common tasks, while user-written code can address specific or custom requirements.

For example, while Python offers built-in functions for data manipulation and mathematical operations, a user might write their own algorithm for sorting a specific set of data based on custom criteria, such as sorting a list of objects based on a particular attribute.

Example of Combining Pre-Written and User-Written Code:

    # Pre-written code: Using the built-in 'sorted' function to sort a list
    numbers = [3, 1, 4, 1, 5, 9]
    sorted_numbers = sorted(numbers)  # Uses pre-written code for sorting

    # User-written code: Sorting based on custom criteria (e.g., sorting in descending order)
    def custom_sort(lst):
        return sorted(lst, reverse=True)  # Custom logic to reverse the order

    custom_sorted_numbers = custom_sort(numbers)  # Uses both pre-written and user-written code
    print(custom_sorted_numbers)  # Output: [9, 5, 4, 3, 1, 1]

In the example above, the sorted() function is a built-in Python function, and the custom_sort() function is user-written to apply a specific sorting order. The combination of both showcases the power of leveraging pre-written code while still writing custom solutions when needed.

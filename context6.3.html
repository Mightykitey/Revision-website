6.3 - Data Systems

6.3.1 - Understand the features and functions of data systems and their importance to organisations:

Data systems are essential tools for organizations to manage, process, and derive value from the vast amounts of data they collect. They play a crucial role in streamlining operations and decision-making. Below are the key features and functions that define modern data systems:

  • Data Wrangling: This refers to the process of cleaning, structuring, and enriching raw data into a usable format. Data wrangling involves:
    • Structure: Organizing data into consistent formats for easier analysis.
    • Clean: Removing errors, inconsistencies, or duplicate entries from data.
    • Enrich: Adding relevant context or data from external sources to enhance the dataset.
    • Validate: Ensuring the data is accurate and conforms to predefined rules or standards.
    • Output: Preparing the data for further analysis or integration into business processes.
  • Core Functions of Data Systems: The following are the fundamental tasks that data systems handle:
    • Input: The process of capturing or importing data from various sources into the system.
    • Search: The ability to retrieve specific data based on queries, often through indexing.
    • Save: Storing data securely and efficiently for future access and analysis.
    • Integrate: Combining data from various sources to create a unified dataset or enable cross-platform analysis.
    • Organize (Index): Structuring data for easy access and efficient querying, often using indexing or database schemas.
    • Output: Delivering processed data or insights to end-users or systems for further action.
    • Feedback Loop: Continuously refining data collection, processing, and analysis based on user feedback and new insights.
  • Data Entry and Maintenance: Data systems need to be maintained to ensure that data is accurate, up-to-date, and easy to input. Challenges include:
    • Online Data Entry: The process of entering data through digital interfaces, often in real-time. Common in e-commerce, customer support, and content management.
    • Risk of Data Entry Errors: Human or system errors during entry can lead to inaccurate data, which may affect decision-making and reporting.
    • Time to Create Data Entry Screens: Designing user-friendly, efficient data entry forms is critical to reducing errors and time spent entering data.
  • Visualisation: Data visualization is the graphical representation of information and data, making it easier to understand and analyze. Common forms include:
    • Graphs/Charts: Line graphs, bar charts, pie charts, etc., for visualizing trends, comparisons, and distributions.
    • Data Tables: Structured tables to organize data for easy reading and reference.
    • Reports: Summary documents that interpret data and provide actionable insights.
    • Infographics: Visual representations combining data with graphic design to convey complex information clearly and attractively.

6.3.2 - Understand the purpose of business information tools and their use in business:

Business information tools enable organizations to analyze data and generate insights that drive informed decision-making. These tools enhance operational efficiency and strategic planning. Some key business information tools include:

  • Business Intelligence (BI) Software: Tools like Power BI, Tableau, and Qlik Sense that allow businesses to aggregate, analyze, and visualize data from various sources to make informed decisions.
  • Customer Relationship Management (CRM) Systems: Platforms like Salesforce and HubSpot that track and manage customer interactions, helping businesses improve sales, customer service, and marketing efforts.
  • Financial Planning Systems: Tools for managing budgets, forecasting, and financial reporting, which help organizations optimize their financial resources and maintain financial health.

By using these tools, organizations can improve efficiency, track performance, and respond more effectively to market changes.

6.3.3 - Understand the features of different data models and how organisations use them to organise data:

Data models are frameworks that describe how data is structured and related within a system. Different models are used depending on the complexity of the data, the relationships between entities, and the intended use of the system. Here are the main types of data models:

  • Conceptual Data Model: A high-level model that defines the data entities and their relationships but does not go into technical detail. It is used to understand the overall structure of the data without focusing on implementation.
  • Logical Data Model: A more detailed model that outlines the structure of the data, such as tables, fields, and relationships. It focuses on how data is logically organized but does not define how it will be physically stored.
  • Physical Data Model: The most detailed model, which specifies how data will be stored in a database system. It includes information about indexing, data types, and storage structures.
  • Hierarchical Database Model: A model where data is organized into a tree-like structure, where each record has a single parent. It is used in legacy systems and is suitable for applications with a clear, hierarchical relationship (e.g., organizational charts).
  • Relational Model: The most common data model used in modern databases, where data is organized into tables with rows and columns. Relationships are defined using keys (primary and foreign). This model is used in relational database management systems (RDBMS) like MySQL, PostgreSQL, and SQL Server.

Choosing the appropriate data model depends on the complexity of the data and the system's needs. A conceptual model is useful in the early stages of system design, while a physical model is necessary for database implementation.

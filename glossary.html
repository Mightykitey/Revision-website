Glossary of Key Terms

Abstraction

The process of simplifying complex systems by focusing on essential characteristics and ignoring unnecessary details.

Acceptable Use

Refers to the policies or guidelines set by an organization that dictate the appropriate use of its resources, particularly when using the internet or network services.

Ad Hoc Network

A temporary network setup created on the fly between devices without requiring a central server or network infrastructure.

Algorithm

A step-by-step procedure or formula for solving a problem or accomplishing a task, typically in computing or mathematics.

Array

A data structure consisting of a collection of elements, each identified by an index or key.

Autonomous Operation

A system's ability to perform tasks or functions without human intervention.

Backbone

The main supporting structure of a computer network, providing the primary pathways for data transfer.

Barcode

A machine-readable code in the form of a series of vertical lines of varying widths, used for identifying products.

Beta Testing

The phase in software development where a product is tested by a select group of users to identify bugs and improve functionality before the final release.

Big Data

Extremely large data sets that are complex and difficult to process with traditional data processing applications.

Bot

A software application that runs automated tasks over the internet, often for repetitive tasks like data scraping or spamming.

Botnet

A network of computers infected with malicious software and controlled as a group, often used for distributed denial-of-service (DDoS) attacks.

Checksum

A value used to verify the integrity of data by calculating the sum of its components and comparing it with the expected result.

Confidentiality

The principle that information is accessible only to those authorized to view it, ensuring privacy and security.

Constant

A value that, once defined, does not change during the execution of a program.

Computational Thinking

A problem-solving process that includes decomposing problems, recognizing patterns, abstracting relevant details, and designing algorithms.

Data Controller

The entity responsible for determining how and why personal data is processed.

Data Mining

The practice of analyzing large sets of data to discover patterns, correlations, or useful information.

Data Packet

A unit of data transmitted over a network that includes both the data and metadata for routing and identification.

Data Subject

An individual whose personal data is collected, stored, or processed.

Decomposition

The process of breaking a complex problem or system into smaller, more manageable components.

Debug

The process of identifying, analyzing, and fixing bugs or errors in software or hardware.

Decapsulation

The process of removing headers and other encapsulation data from a packet to retrieve the original data inside.

Ethical

Conforming to accepted standards of conduct in technology development, such as respecting privacy, fairness, and responsibility.

Fetch-Execute Cycle

The basic cycle in a computer's CPU that fetches an instruction from memory, decodes it, and executes it.

Global Variable

A variable that is accessible throughout the entire program, often affecting multiple functions or components.

GPS

Global Positioning System, a satellite-based system for determining the location of a device or person on Earth.

Hacking

The act of gaining unauthorized access to computer systems or networks to exploit vulnerabilities.

Hacktivists

Individuals or groups who use hacking techniques to promote political or social causes.

Heat Sink

A device used to dissipate heat from a component, typically found in electronics and computers.

HTML

HyperText Markup Language, the standard language used to create and structure content on the web.

Hypervisor

A software that enables multiple virtual machines to run on a single physical machine.

ICO

Initial Coin Offering, a fundraising mechanism in which new cryptocurrencies are sold to investors.

Identifier

A name used to identify a variable, function, or other entity in a program.

Increment

The process of increasing a variable by a specified amount, typically by one.

Index

The position of an element within an array or list, often used to access or reference specific data.

Iteration

Repeating a process or set of instructions in a loop until a certain condition is met.

Layering

The practice of organizing complex systems into distinct levels that interact with one another, often used in network protocols.

List

A data structure that stores an ordered collection of elements, which can be accessed by their position in the list.

Loop

A control structure that repeats a set of instructions based on a specified condition.

Messy Problems

Problems that are complex, disorganized, and difficult to solve due to ambiguity or missing information.

Metaverse

A collective virtual shared space, typically consisting of interconnected 3D virtual environments.

Metrics

Quantitative measures used to assess performance, quality, or efficiency in various contexts like software development or data analysis.

Middleware

Software that acts as a bridge between different applications or systems, enabling communication and data management.

Modularization

The practice of breaking a system or problem into smaller, independent modules that can be worked on separately, improving manageability.

Moral

Relating to principles of right and wrong behavior, often used in discussions of ethical issues in technology.

Node

An individual device or point in a network, often responsible for routing or managing data.

Open-Source

Software that is made freely available for modification and redistribution, usually with a community-based development model.

Opt-in/Opt-out

The ability to choose whether or not to participate in a service or offer, often related to privacy and data collection preferences.

Packet Switching

A method of transmitting data over a network by breaking it into small packets that are sent independently and reassembled at the destination.

Parameter

A variable that is used to pass information to functions or procedures in a program.

Patent

A legal right granted to an inventor, giving them the exclusive right to make, use, or sell their invention for a specified time period.

Pattern Recognition

The ability to identify patterns and trends within data, which can help with predictions and problem-solving.

Persona

A fictional character created to represent a user type, often used in design and user experience research to guide decisions.

Privacy

The right of individuals to control the collection and use of their personal information.

Proprietary

Software, technology, or intellectual property that is owned by a specific entity and not freely available for public use.

Protocols

Rules and conventions for communication between networked devices, ensuring that data can be transferred reliably and securely.

Pseudocode

A simplified, informal way of representing algorithms, using plain language and basic programming concepts.

QR Code

A type of matrix barcode that can store information such as URLs, text, or contact details, readable by a scanner or smartphone.

Reserved Word

Words in a programming language that have a predefined meaning and cannot be used as identifiers (e.g., `if`, `else`, `while`).

RFID Tags

Radio Frequency Identification tags used to store and transmit data through radio waves, often for tracking purposes.

Robust

Referring to a system's ability to handle errors or unexpected situations without failure.

Scope

The region within a program where a variable or function can be accessed or modified.

Script

A set of instructions written in a scripting language to automate tasks or control applications.

Selection

A control structure in programming where a decision is made based on a condition (e.g., `if`, `switch`).

Sequence

The order in which instructions are executed in a program or algorithm.

Social

Refers to the interactions and relationships between individuals or groups, particularly in online or networked environments.

Solvent

A substance that dissolves another substance, often used in chemistry or solution-based processes.

Stakeholder

An individual or group that has an interest in a project's outcome, such as users, developers, or investors.

Subroutine

A set of instructions that performs a specific task, often used as a reusable component within a larger program.

Syn

Short for "synchronize," often used in computing to ensure processes or actions occur in a specific order.

Syntax

The rules and structure of a programming language that dictate how code should be written and formatted.

Syntax Error

An error in a program that occurs when the code does not follow the correct syntax of the programming language.

Tangible

Something that can be touched or physically interacted with, as opposed to something abstract.

Trace Table

A tool used in programming to manually track the values of variables at each step of an algorithm's execution.

Transcription Error

An error that occurs when data or text is copied incorrectly.

Transposition Error

An error where elements in data are swapped or reversed incorrectly.

URI

Uniform Resource Identifier, a string of characters used to identify a resource on the internet.

Validation

The process of checking if data is accurate, complete, and meets predefined criteria before processing or storage.

Variable

A named storage location in a program that can hold different values during its execution.

Verification

The process of ensuring that a system, product, or piece of data meets specific requirements or standards.

Visualization

The process of representing data or concepts in graphical or visual formats to aid understanding.

White Goods

Large home appliances like refrigerators, washing machines, and ovens, typically in white or stainless steel finishes.
